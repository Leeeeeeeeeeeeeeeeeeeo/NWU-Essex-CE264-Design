## 🚀 项目简介

这是 CE264 课程的数字电路设计项目，具体实现路线如下：

**设计路线**：  
使用 Multisim 设计电路原理图 → 使用 Multisim 将原理图文件转换为 VHDL 文件 → 使用 Vivado 编译 VHDL → 将编译结果下载到 Basys3 开发板实现设计。

## 🛠️ 如何使用

> 前提：完成第一课 Tutorial，将对这一套设计到实现的流程有所了解。

### Multisim 原理图 (`.ms14`/`.mp14`)

*   大部分文件属于顶层项目，另有少部分额外的设计。
*   Multisim 会从顶层项目文件读取文件路径，因此直接打开 `project` 里的 `.mp14` 文件可能会报告找不到路径。**没关系，对着 `circuit` 里面的名字找即可**。

### VHDL 文件 (`.vhd`)

*   文件可能会被系统识别为某种压缩包格式，但没有影响。
*   同一个命名的主文件和 `_pkg` 文件配套使用。

### 约束文件 (`.xdc`)

*   别看这玩意的内容多，其实操作很简单：**找到你需要的引脚 `PIN`，取消注释、修改 `port` 名称**即可。

## 💡 一些感悟

*   本课程的设计路线与常规数字系统设计不同：
    *   **本课程**：从较底层的逻辑门开始，属于**门级描述**。
    *   **常规路线**：多为寄存器传输级描述（RTL），更底层的门级电路由设计软件自动生成。
*   因此，本课程着重于通过逻辑门、时序逻辑和组合逻辑的设计，实现寄存器间的数据流动与处理，进而达成行为级和系统级的设计。
*   **本课程的重心在第一步——原理图设计上**。设计完成之后的所有步骤，都可以类比成嵌入式开发中的“程序编译与烧录”过程。
*   也就是说，你会有 **80% 以上的时间在和 Multisim 打交道**，对着密密麻麻的线路修改原理图。

## ⚠️ 写在最后

1.  **性能警告**：完全体原理图不算小，在低配电脑上运行 Multisim 可能会掉帧，甚至有卡死的风险。
2.  **玄学因素**：从原理图到开发板实现，这一路上变量太多，导致能否成功实现带点玄学因素。
3.  **显示问题**：有时候即使是完全没问题的原理图，导出编译后的文件在开发板上显示效果也会有问题（例如多出一条缝、莫名其妙消失的块等）。
    *   **建议**：排查完错误后，可以尝试在原理图里**拔掉几根导线，再接回去，然后重新导出 VHDL**。（我自己的 `stable_edition` 导出了二十次才成功。）
4.  **Vivado 报错**：在某些型号电脑上 vivado 会报错（使用IU的Y9000P和幻16都坠机了，但AU的小新Pro不在此列，唉这AMD怎么这么坏啊😡），卡在 `run synthesis` 这一步，显示综合完成但是文件系统错误。
